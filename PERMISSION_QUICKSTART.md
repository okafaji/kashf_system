# ⚡ البدء السريع - إدارة الصلاحيات (5 دقائق)

## 1️⃣ الوصول إلى صفحة الصلاحيات (10 ثوانٍ)

```
URL: http://localhost:8000/admin/permissions
```

**متطلبات**:
- يجب أن تكون مسجل دخول كـ admin أو لديك صلاحية manage-settings

---

## 2️⃣ إعطاء مستخدم صلاحية التعديل (30 ثانية)

### الخطوات:
1. **الذهاب إلى** `/admin/permissions`
2. **البحث عن** المستخدم في الجدول
3. **النقر على** زر "تعديل" (✏️)
4. **في صفحة التعديل**, ضع علامة على `edit-payrolls` ✓
5. **النقر** زر "حفظ"
6. ✅ **تمام!** المستخدم يمكنه الآن تعديل الكشوفات

---

## 3️⃣ إعطاء مستخدم دور Admin (الحد الأدنى 1 دقيقة)

### الخطوات:
1. `/admin/permissions` → البحث والنقر تعديل
2. ضع علامة على `☐ admin` تحت الأدوار
3. النقر حفظ
4. ✅ المستخدم أصبح admin كامل!

---

## 4️⃣ الصلاحيات الأساسية

| التصنيف | الصلاحية | الوصف |
|--------|---------|-------|
| **عرض** | view-payrolls | رؤية قائمة الكشوفات |
| **عرض** | view-employees | رؤية قائمة الموظفين |
| **إنشاء** | create-payrolls | إنشاء كشوفات جديدة |
| **تعديل** | **edit-payrolls** | **✏️ تعديل الكشف الموجود** |
| **حذف** | delete-payrolls | حذف كشف |
| **طباعة** | print-payrolls | طباعة الكشوفات |
| **إدارة** | manage-users | إدارة المستخدمين |
| **إدارة** | manage-settings | إعدادات النظام |

---

## 5️⃣ أمثلة سريعة

### مثال 1: مستخدم محاسب (يرى فقط)
```
☑ view-payrolls
☐ create-payrolls
☐ edit-payrolls
☐ delete-payrolls
```

### مثال 2: مستخدم مدير (يرى ويعدل)
```
☑ view-payrolls
☑ create-payrolls
☑ edit-payrolls
☑ delete-payrolls

```

### مثال 3: مستخدم admin (يفعل كل شيء)
**اختر الدور**: ☑ admin

## 6️⃣ التحقق السريع

### يمكنك التحقق بـ:
```bash
php artisan tinker
>>> User::find(2)->can('edit-payrolls')
=> true  ✓ لديه الصلاحية
=> false ✗ لا يملكها
```

---

## 7️⃣ حل المشاكل السريع

| المشكلة | الحل |
|--------|------|
| خطأ 403 | تأكد من صلاحية manage-users |
| لا أرى الأزرار | امسح cache: `php artisan cache:clear` |
| التغييرات لم تطبق | سجل الدخول من جديد أو افتح صفحة جديدة |
| لا أجد صفحة الصلاحيات | تأكد من: آدرس صحيح + مسجل دخول + صلاحية صحيحة |

---

## 8️⃣ الروتس المسجلة

```bash
# عرض قائمة المستخدمين
GET  /admin/permissions

# تعديل مستخدم معين
GET  /admin/permissions/{user-id}
PUT  /admin/permissions/{user-id}

# عمليات سريعة (AJAX)
POST /admin/permissions/{user-id}/grant
POST /admin/permissions/{user-id}/revoke
POST /admin/permissions/{user-id}/assign-role
POST /admin/permissions/{user-id}/remove-role
```

## 9️⃣ كود الـ Blade للتحقق

إذا أردت التحقق يدويًا في الـ view:

```blade
@can('edit-payrolls')
    <!-- المستخدم الحالي لديه صلاحية -->
    <button>تعديل</button>
@endcan

@if(auth()->user()->hasRole('admin'))
    <!-- فقط الـ admins -->
    <button>حذف كل شيء</button>
@endif
```

## 🔟 أسئلة شائعة

**س: كيف أعطي صلاحية لـ 10 مستخدمين؟**  
جـ: اذهب إلى حر واحد ومكرر الخطوات 10 مرات (أم أطلب Bulk assignment في المستقبل)

**س: كيف أزيل صلاحية؟**  
جـ: اذهب لصفحة التعديل وأزل العلامة ثم احفظ

**س: هل الصلاحيات تُطبق فوراً؟**  
جـ: بعد الحفظ مباشرة، لكن قد يحتاج المستخدم تحديث الصفحة

**س: ماذا لو نسيت كلمة المرور؟**  
جـ: هذا موضوع آخر 😄 (انظر إلى صفحة Reset Password)


## 📞 هل تحتاج مساعدة؟

- **خطأ في الصفحة**: شوف `storage/logs/laravel.log`
- **لا تظهر الأزرار**: امسح cache وحدّث الصفحة
- **صلاحيات خاطئة**: تأكد أنك في صفحة الصلاحيات الصحيحة
