4. **موظف عادي**: يرى الكشوفات التي أنشأها فقط

### 🔧 كيفية الاستخدام والاختبار

#### 1. التحضيرات الأساسية

**تعيين الأدوار للمستخدمين:**
```php
// في tinker أو أي سيد
$user = User::find(1);  // المستخدم المراد
$user->assignRole('رئيس قسم');  // أو مسؤول وحدة / مسؤول شعبة
```

**تعيين القسم للمستخدمين:**
```php
$user->update(['department_id' => 1]); // رقم القسم
```

#### 2. التشغيل والاختبار

**قبل التجربة - تأكد من:**
- [ ] النظام يعمل (في development server)
- [ ] قاعدة البيانات تحتوي على كشوفات
- [ ] المستخدمة لديه دور محدد
- [ ] المستخدم لديه department_id

**اختبر لوحة التحكم:**
1. افتح `/dashboard`
2. تحقق من ظهور تابين: "احصائياتي" و "احصائيات الفريق"
3. اختر تاريخ مختلف وتأكد أن الفلاترات تعمل بشكل مستقل
4. انقل بين التابين للتأكد من عمل التحميل

#### 3. اختبار API (بواسطة Postman أو curl)

```bash
# الحصول على السنوات المتاحة
curl http://localhost/api/user-years \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "X-Requested-With: XMLHttpRequest"

# الحصول على الأشهر للسنة 2026
curl http://localhost/api/user-months?year=2026 \
  -H "Authorization: Bearer YOUR_TOKEN"

# الحصول على الأيام لشهر معين
curl http://localhost/api/user-days?year=2026&month=3 \
  -H "Authorization: Bearer YOUR_TOKEN"

# الحصول على الإحصائيات
curl "http://localhost/api/user-stats?year=2026&month=3&day=5" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 🚨 ملاحظات مهمة

1. **جميع الكشوفات الجديدة** ستحتوي على:
   - `user_id`: معرف المنشيء
   - `created_by_department_id`: QS م القسم الذي أنشأ الكشف

2. **الكشوفات القديمة** قد يكون لديها `NULL` في هذه الحقول
   - سيتم عرضها للأدمن فقط (رئيس قسم)
   - موظفين عاديين لن يروا الكشوفات القديمة بدون user_id

3. **الهيكل الهرمي للأقسام**:
   ```
   ديوان عام (id=1, parent_id=NULL)
   ├── شعبة 1 (id=2, parent_id=1)
   │   ├── وحدة 1 (id=3, parent_id=2)
   │   └── وحدة 2 (id=4, parent_id=2)
   └── شعبة 2 (id=5, parent_id=1)
   ```
