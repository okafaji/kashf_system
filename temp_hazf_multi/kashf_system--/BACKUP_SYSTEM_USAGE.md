# نظام النسخ الاحتياطية - دليل الاستخدام

## نظرة عامة
نظام متكامل لإنشاء نسخ احتياطية كاملة من قاعدة البيانات والأكواد، مع إمكانية تحميلها وحذفها وإعادتها بسهولة.

---

## الميزات الرئيسية

✅ **نسخ احتياطية كاملة**
- قاعدة البيانات (mysqldump)
- الأكواد والملفات (ZipArchive)
- في نسخة واحدة مع طابع زمني فريد

✅ **إدارة سهلة**
- واجهة رسومية في لوحة التحكم
- إنشاء نسخة جديدة بضغطة زر
- تحميل النسخ القديمة
- حذف النسخ غير المستخدمة

✅ **أمن البيانات**
- صلاحيات محدودة (فقط Admin)
- تسجيل جميع العمليات في Logs
- حذف آمن للملفات

---

## متطلبات النظام

### قبل الاستخدام، تأكد من وجود:
```
1. mysqldump مثبت (يأتي مع MySQL/MariaDB)
2. PHP ZipArchive extension مفعّل
3. صلاحيات كتابة على مجلد storage/backups
4. أن المستخدم يمتلك دور Admin
```

### التحقق من المتطلبات:
```bash
# التحقق من mysqldump
mysqldump --version

# التحقق من ZipArchive
php -i | grep zip
```

---

## التثبيت والإعداد

### 1. تشغيل Migration (إن وجدت)
```bash
php artisan migrate
```

### 2. إنشاء Permissions والـ Roles
```bash
php artisan db:seed --class=BackupPermissionSeeder
```

هذا الأمر سيقوم بـ:
- إنشاء permission `manage-backups`
- إسناده إلى دور Admin

### 3. التحقق من إنشاء المجلد
```bash
# يجب أن يكون موجود storage/backups
ls -la storage/backups
```

إذا لم يكن موجوداً، أنشئه يدويا:
```bash
mkdir -p storage/backups
chmod 755 storage/backups
```

## آلية العمل

### JavaScript (backup-manager.js)

```javascript
// 1. تحميل النسخ عند فتح الصفحة
loadBackupsList()
  ↓
GET /backups/list
  ↓
عرض النسخ في الجدول

// 2. إنشاء نسخة جديدة
createBackup()
  ↓
POST /backups/create
  ↓
✅ عرض الحالة (جاري/نجح/فشل)
  ↓
تحديث قائمة النسخ

// 3. تحميل نسخة
downloadBackup(timestamp)
  ↓
GET /backups/download/{timestamp}
  ↓
تنزيل ملف ZIP

// 4. حذف نسخة
deleteBackup(timestamp)
  ↓
DELETE /backups/delete/{timestamp}
  ↓
تحديث القائمة
```

### PHP (BackupController.php)

```php
// المسارات
POST /backups/create       → createBackup()
GET  /backups/list         → listBackups()
GET  /backups/download     → downloadBackup()
DELETE /backups/delete     → deleteBackup()

// الملفات المُنشأة
storage/backups/
  ├── backup_2024_01_15_10_30_45/
  │   ├── database_2024_01_15_10_30_45.sql
  │   └── code_2024_01_15_10_30_45.zip
  └── backup_2024_01_14_15_20_10/
      ├── database_2024_01_14_15_20_10.sql
      └── code_2024_01_14_15_20_10.zip
```

## استخدام الواجهة الرسومية

### 1. الوصول إلى النسخ الاحتياطية
```
الصفحة الرئيسية (Dashboard)
  ↓
البحث عن قسم "النسخ الاحتياطية" (الأسفل)
```

### 2. إنشاء نسخة جديدة
```
1. انقر على زر "💾 إنشاء نسخة احتياطية"
2. سيظهر نموذج تأكيد
3. انقر على "إنشاء نسخة احتياطية"
4. انتظر حتى ينتهي الإنشاء
   (يعتمد على حجم قاعدة البيانات)
5. ستظهر الرسالة "✅ تم إنشاء النسخة بنجاح"
```

### 3. عرض النسخ الموجودة
```
1. قائمة النسخ تُظهر:
   - التاريخ والوقت
   - عدد الملفات
   - حجم النسخة (بالـ MB)

2. الترتيب: الأحدث أولاً
```

### 4. تحميل نسخة
```
1. اضغط على زر "تحميل" بجانب النسخة
2. سيبدأ تحميل ملف ZIP
3. الملف يسمى: kashf_backup_TIMESTAMP.zip
```

### 5. حذف نسخة
```
1. اضغط على زر "حذف" بجانب النسخة
2. تأكيد الحذف
3. ستُحذف النسخة نهائياً
```

## استخدام API (للمطورين)

### الحصول على قائمة النسخ
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
     https://example.com/backups/list
```

**الرد:**
```json
{
  "success": true,
  "backups": [
    {
      "folder": "backup_2024_01_15_10_30_45",
      "timestamp": "2024_01_15_10_30_45",
      "date": "2024/01/15 10:30:45",
      "size": "150.45 MB",
      "files": 2
    }
  ]
}
```

### إنشاء نسخة جديدة
```bash
curl -X POST \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "X-CSRF-TOKEN: YOUR_CSRF_TOKEN" \
     https://example.com/backups/create
```

**الرد:**
```json
{
  "success": true,
  "message": "تم إنشاء النسخة الاحتياطية بنجاح",
  "timestamp": "2024_01_15_10_30_45",
  "backup_folder": "/path/to/backup_folder"
}
```

### تحميل نسخة
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
     https://example.com/backups/download/2024_01_15_10_30_45 \
     -o backup.zip
```

### حذف نسخة
```bash
curl -X DELETE \
     -H "Authorization: Bearer YOUR_TOKEN" \
     -H "X-CSRF-TOKEN: YOUR_CSRF_TOKEN" \
     https://example.com/backups/delete/2024_01_15_10_30_45
```

## استكشاف الأخطاء

### خطأ: "فشل إنشاء النسخة الاحتياطية"
```
السبب المحتمل: mysqldump غير مثبت أو غير متوفر في PATH
الحل:
1. تحقق من تثبيت MySQL
2. أضف MySQL إلى متغيرات النظام (PATH)
3. جرب الأمر: mysqldump --version
```

### خطأ: "فشل جلب قائمة النسخ"
```
السبب المحتمل: مجلد storage/backups غير موجود
الحل:
1. أنشئ المجلد يدويا
2. تأكد من صلاحيات الكتابة
3. أعد تشغيل الصفحة
```

### خطأ: "فشل ضغط الكود"
```
السبب المحتمل: ZipArchive غير مفعّل
الحل:
1. سجل دخولك كمسؤول
2. عدّل php.ini وأضف: extension=zip
3. أعد تشغيل الخادم
4. تحقق: php -m | grep zip
```

### خطأ: "النسخة الاحتياطية غير موجودة"
```
السبب المحتمل: تم حذف المجلد أو النسخة قديمة جداً
الحل:
1. تحقق من قائمة النسخ المتاحة
2. تأكد من التاريخ الصحيح
3. قد تكون النسخة قد حُذفت بالفعل
```

### الخطأ: "محظور الوصول"
```
السبب المحتمل: المستخدم لا يمتلك دور Admin
الحل:
1. تحقق من دور المستخدم
2. امنح دور Admin إذا لزم الأمر
3. تأكد من تشغيل BackupPermissionSeeder
```

## أمان البيانات

### نقاط الأمان المُطبقة:
- ✅ فقط المستخدمون بدور Admin يمكنهم إنشاء نسخ
- ✅ كل عملية يتم تسجيلها في Logs
- ✅ الملفات لا تُحذف بشكل عرضي (طلب تأكيد)
- ✅ معرفات النسخ فريدة (timestamp)

### التوصيات:
1. **نسخ منتظمة**: أنشئ نسخة يومية على الأقل
2. **حفظ خارجي**: نزّل النسخ إلى جهاز خارجي
3. **فحص دوري**: اختبر النسخ الاحتياطية بشكل دوري
4. **توثيق العمليات**: احتفظ بسجل للنسخ الهامة

## أداء وحدود النظام

### حجم النسخة المتوقع:
```
قاعدة بيانات: 50-200 MB
الأكواد والملفات: 100-500 MB
الكل مع الضغط: 50-150 MB
```

### المدة الزمنية:
```
صغير (< 100 MB):  30 ثانية
متوسط (100-500 MB): 1-3 دقائق
كبير (> 500 MB):  3-10 دقائق
```

### حد أقصى للنسخ المخزنة:
```
بدون حد أقصى محدد
نوصي بحذف النسخ التي مضى عليها أكثر من 30 يوم
```
